# 项目规则

## 📋 用途

本文件定义项目的通用规则和约束，适用于整个项目。

## 🔒 提交约束

- **自动提交**：工作过程中不要自动提交代码
- **手动提交**：只有在收到明确的 "提交代码" 指令后，才执行 `git add`、`git commit` 和 `git push`
- **提交信息**：使用清晰的提交信息格式，如 `[类型] 描述信息`
- **分支管理**：遵循标准的分支命名规范

## 📁 目录结构

```
项目根目录/
├── .trae/           # Trae 智能体相关文件
│   ├── agents/      # 智能体定义
│   ├── skills/      # 技能定义
│   ├── prompts/     # 提示词模板
│   └── rules/       # 规则文件
├── 会议纪要/         # 会议纪要文件
├── AI实战/          # AI 相关文章
├── 读书分享/         # 读书分享文章
├── 生活观察/         # 生活观察文章
└── 归档/            # 归档文件
```

## 📄 文件命名规范

- **智能体文件**：`[智能体名称].md`
- **技能文件**：`[技能名称].skill`
- **规则文件**：`[规则类型]_rules.md`
- **提示词文件**：`[功能名称].prompt`

## 🔍 代码规范

- **编码风格**：保持一致的编码风格
- **代码质量**：确保代码可读性和可维护性
- **注释**：适当添加注释，提高代码可理解性
- **测试**：确保代码通过测试

## 🔒 安全规范

- **敏感信息**：不提交敏感信息到代码仓库
- **权限管理**：合理设置文件和目录权限
- **数据安全**：保护用户数据和隐私

## 🎯 目标

通过标准化的项目管理流程，确保：
- 代码质量稳定
- 工作流程规范
- 团队协作高效
- 项目进展顺利
