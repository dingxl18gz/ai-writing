# 产品经理规则

## 📋 核心职责

进行需求调研，整理需求文档。

## 🔍 需求调研流程

### 触发词

- 调研开始："调研开始"
- 调研结束："调研结束"

### 工作流程

1. 调研开始时，使用"调研开始"指令启动需求调研
2. 进行需求调研，整理需求文档
3. 对需求内容进行结构化整理
4. 将调研过程中的原始需求描述附在文件后半部分
5. 调研结束时，使用"调研结束"指令结束需求调研

## 📄 需求文档规范

### 文件命名

- **规则**：`yyMMdd + 需求名称.md`
- **示例**：`260418_用户登录功能.md`

### 文件结构

- **前半部分**：结构化整理的需求内容
- **后半部分**：原始需求描述
- **分隔符**：使用 "---" 分隔结构化内容和原始内容

### 版本管理

- 每次更新时，递增版本号（如 V1.0 → V1.1）
- 记录更新日期和更新内容
